 Add Change Password & Delete Account
What & Why
Add two new features to the Settings screen for both the mobile and web versions of the app:

Change Password — lets a logged-in user update their password directly from Settings without going through the forgot-password flow.
Delete Account — lets a user permanently delete their account with a confirmation step.
Both features must follow the existing MVC/GetX pattern (data source → controller → view) and match the current UI style of each platform.

Done looks like
Mobile Settings screen has two new tiles under the "الحساب" section: "تغيير كلمة المرور" and "حذف الحساب".
Web Settings page has a new card titled "الأمان" containing the same two actions.
Tapping "تغيير كلمة المرور" opens a bottom-sheet (mobile) or dialog (web) with fields for current password, new password, and confirm new password; on success a snackbar confirms the change.
Tapping "حذف الحساب" opens a confirmation dialog matching the logout-confirm style; the user must type a confirmation word or tap a destructive button; on success the session is cleared and the user is redirected to login.
All loading states are shown (button spinner / disabled state) while requests are in flight.
No backend errors are swallowed — meaningful Arabic error messages are shown via snackbar.
Out of scope
Forgot/Reset password flow (already exists).
Email change or profile-data edits.
Admin-side account recovery.
Steps
API endpoints — Add two new getters to AppLink in lib/linkapi.dart:

changePassword → $server/auth/change-password
deleteAccount → $server/auth/delete-account
Remote data sources — Create two new files under lib/data/sourcedata/remote/Auth/:

ChangePasswordData.dart: a class that takes Crud and exposes changePassword({currentPassword, newPassword, newPasswordConfirmation}) calling crud.postData.
DeleteAccountData.dart: a class with deleteAccount() calling crud.deleteData (or postData based on backend convention) on the delete-account endpoint.
SettingsController — new methods — In lib/controller/Home/settings_controller.dart:

Instantiate ChangePasswordData and DeleteAccountData (using the existing Crud() pattern).
Add observable fields: isChangingPassword, isDeletingAccount.
Add changePassword({required String current, required String newPass, required String confirm}): calls the data source, shows success/error snackbar, closes the dialog on success.
Add deleteAccount(): calls the data source, then calls Services.clearSession() and navigates to AppRoutes.LOGIN on success, or shows an error snackbar on failure.
Mobile view — Change Password bottom-sheet — In lib/view/screen/Home/settings/settings_view.dart:

Add a new tile "تغيير كلمة المرور" (icon: lock_outline) in the "الحساب" section, before or after the existing "معلومات الحساب" tile.
On tap, call a helper that shows a showModalBottomSheet containing a form with three TextFormFields (current password, new password, confirm password) and a submit button; the submit button calls controller.changePassword(...). Match the existing card/tile colour and border-radius style.
Mobile view — Delete Account tile — In the same settings_view.dart:

Add a destructive tile "حذف الحساب" (icon: delete_forever_outlined, isDestructive: true) in the "الحساب" section or below the logout tile.
On tap, show an AlertDialog styled like the existing logout confirm dialog (red icon, cancel + confirm buttons) calling controller.deleteAccount().
Web view — Security card — In lib/web/view/pages/web_settings_page.dart:

Add a new _card('الأمان', [...]) section between the Notifications card and the Logout button.
Inside the card, add two action rows:
"تغيير كلمة المرور" (icon: lock_outline): tapping calls _showChangePasswordDialog().
"حذف الحساب" (icon: delete_forever_rounded, red): tapping calls _confirmDeleteAccount().
_showChangePasswordDialog(): Get.dialog(...) with the same WebTheme-styled dialog shape as the logout dialog, containing a Form with three password fields and a submit button wired to c.changePassword(...).
_confirmDeleteAccount(): Get.dialog(...) following the exact same structure as _confirmLogout() but with a delete icon and c.deleteAccount() call.
